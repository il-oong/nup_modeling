# NUP Modeling - 기획서 (Blender 애드온)

## 1. 프로젝트 개요

- **프로젝트명**: NUP Modeling
- **목적**: AI 에이전트 체인이 대화하며 Blender 3D 모델링 코드를 디벨롭하는 Blender 애드온
- **AI 모델**: Google Gemini Flash 3 Preview
- **API 키**: Blender 애드온 설정(Preferences)에서 등록
- **핵심 기능**:
  1. 에이전트 체인 → 자동으로 코드 디벨롭
  2. 대화형 모델링 → 사용자가 대화하며 모델을 수정/발전
  3. 아웃풋 지정 → 원하는 결과물(형태, 포맷)을 정해두고 그 방향으로 모델링

---

## 2. 에이전트 체인 구조

### 에이전트 역할

| 에이전트 | 역할 |
|----------|------|
| Architect | 모델링 구조 설계, 파트 분리, 전체 방향 |
| Coder | Blender Python 코드 작성 |
| Tester | 코드를 `exec()`로 실제 실행, 오류 캐치 및 리포트 |
| Reviewer | 결과물 품질 리뷰, 아웃풋 목표와 비교 |
| Optimizer | 메시 최적화, 코드 정리 |

### 체인 흐름

```
사용자: "자동차 모델링 해줘" + 아웃풋 설정 (로우폴리, .fbx)
    ↓
┌─── 체인 라운드 ──────────────────────────────────────┐
│                                                       │
│  Architect: 구조 설계 (차체/바퀴/창문 분리)            │
│      ↓                                                │
│  Coder: 코드 작성 → [v1]                              │
│      ↓                                                │
│  Tester: exec() 실행 → 오류 발생 시 → Coder로 리턴    │
│      ↓ ✅ 통과                                        │
│  Reviewer: "아웃풋 목표(로우폴리)에 비해 폴리곤 많음"   │
│      ↓                                                │
│  Coder: 수정 → [v2]                                   │
│      ↓                                                │
│  Tester: exec() 실행 → ✅ 통과                        │
│      ↓                                                │
│  Optimizer: 메시 정리 + modifier 적용 → [v3]          │
│      ↓                                                │
│  Tester: exec() 실행 → ✅ 통과                        │
│                                                       │
└─── 사용자 확인 ───────────────────────────────────────┘
    ↓
사용자: "앞부분 더 둥글게" (대화형 수정)
    ↓
┌─── 다음 라운드 (대화 반영) ────────────────────────────┐
│  Coder: 피드백 반영 수정 → Tester → Reviewer → ...    │
└────────────────────────────────────────────────────────┘
    ↓
최종 완료 → 아웃풋 포맷으로 자동 내보내기
```

## 3. 주요 기능

### 3.1 에이전트 체인 (자동 디벨롭)
- 사용자가 요청 입력하면 에이전트들이 체인으로 자동 진행
- Tester가 실제 Blender에서 코드 실행 → 진짜 오류 검증
- Tester 실패 시 자동으로 Coder에게 리턴 (최대 재시도 횟수 설정)
- 라운드별 코드 버전 관리

### 3.2 대화형 모델링
- 체인 완료 후 사용자가 대화로 추가 수정 요청
  - "앞부분 더 둥글게"
  - "색상 빨간색으로"
  - "바퀴 크기 줄여"
- 이전 대화 컨텍스트 유지 → 점진적 발전
- 수정 요청마다 Tester 검증 포함
- 3D 뷰포트에서 실시간 결과 확인

### 3.3 아웃풋 설정
- 모델링 시작 전 목표 아웃풋 지정:

| 설정 항목 | 옵션 예시 |
|-----------|-----------|
| 스타일 | 로우폴리, 하이폴리, 카툰, 리얼리스틱 |
| 용도 | 게임 에셋, 렌더링, 3D 프린팅, 애니메이션 |
| 폴리곤 수 | 제한 설정 (예: 최대 10,000) |
| 내보내기 포맷 | .blend, .fbx, .obj, .stl, .glTF |
| 크기/스케일 | 실제 비율, 커스텀 크기 |
| 머티리얼 | 포함/미포함, PBR 여부 |

- Reviewer가 매 라운드마다 아웃풋 목표와 비교하여 피드백
- 완료 시 지정 포맷으로 자동 내보내기

### 3.4 실시간 실행
- Tester가 코드를 Blender에서 직접 `exec()` 실행
- 결과를 3D 뷰포트에서 바로 확인
- 오류 발생 시 에러 로그 표시
- Undo 지원 → 이전 버전으로 롤백 가능

## 4. 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Python |
| Blender API | bpy (Blender Python API) |
| AI SDK | `google-generativeai` (Python) |
| UI | Blender Panel (N-Panel 사이드바) |
| 코드 실행 | `exec()` + `bpy.ops.ed.undo_push()` |
| 내보내기 | `bpy.ops.export_*` |
| 배포 | .zip 애드온 파일 |

## 6. 데이터 흐름

```
사용자 입력 (요청 + 아웃풋 설정)
    ↓
에이전트 체인 시작
    ↓
각 에이전트 → Gemini API 호출 (google-generativeai SDK)
    ↓
Tester → exec()로 Blender 내 실행
    ├── 실패 → 에러 로그 + Coder에게 리턴
    └── 성공 → 다음 에이전트로 진행
    ↓
Reviewer → 아웃풋 목표와 비교
    ├── 미달 → 피드백 + Coder에게 리턴
    └── 충족 → Optimizer로 진행
    ↓
라운드 완료 → 사용자 확인
    ↓
사용자 대화형 수정 (선택)
    ↓
최종 완료 → 아웃풋 포맷으로 내보내기
```

## 7. 에이전트 시스템 프롬프트 (핵심)

각 에이전트는 동일한 Gemini 모델이지만 시스템 프롬프트가 다름:

- **Architect**: "Blender 모델링 설계자. 요청을 분석해 파트별 구조를 설계. 아웃풋 목표: {output_settings}"
- **Coder**: "Blender Python 전문가. bpy API로 모델링 코드 작성. 이전 대화와 피드백 반영. 반드시 실행 가능한 완전한 코드 출력"
- **Tester**: "코드 검증 전문가. 문법 오류, import 오류, bpy API 호환성, 런타임 에러를 체크. 실행 결과 리포트"
- **Reviewer**: "3D 모델링 품질 리뷰어. 아웃풋 목표({output_settings})와 현재 결과를 비교. 구체적 개선점 제안"
- **Optimizer**: "코드 최적화 전문가. 불필요한 vertex 제거, modifier 활용, 코드 정리"

## 9. 보안 고려사항

- API 키는 Blender Addon Preferences에 저장 (로컬)
- 코드 `exec()` 실행 시 안전장치:
  - 위험한 모듈(os, subprocess, sys 등) import 차단
  - 타임아웃 설정
  - bpy 관련 코드만 허용
- 생성된 코드는 사용자 확인 후 실행 옵션 제공
